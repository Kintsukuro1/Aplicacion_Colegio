# Seguimiento: notas, tareas y pruebas

## Pasos a seguir
1. Extender el dominio académico para soportar evaluaciones híbridas.
2. Mantener la entrega por archivo como respaldo y agregar resolución online.
3. Añadir autocorrección básica y estado pendiente de aprobación.
4. Conectar notificaciones para revisión docente y publicación al estudiante.
5. Ajustar API y frontend para crear, responder y aprobar.
6. Validar compatibilidad con los flujos legacy existentes.

## Reglas del proyecto
- No romper los endpoints actuales de tareas, evaluaciones y calificaciones.
- Mantener la entrega por archivo como opción válida.
- La nota no debe publicarse al estudiante sin aprobación docente.
- Reutilizar notificaciones existentes cuando sea posible.
- Preferir cambios pequeños, trazables y compatibles con el flujo actual.
